# 11. Repository Structure

## 1. Цель

Структура репозитория должна поддерживать:
- разделение доменов
- понятную работу агентов
- предсказуемую навигацию
- независимое развитие модулей

---

## 2. Дерево верхнего уровня

```text
/
├─ AGENTS.md
├─ README.md
├─ docs/
├─ apps/
├─ packages/
├─ tests/
├─ scripts/
├─ deploy/
├─ .editorconfig
├─ .gitattributes
├─ .gitignore
└─ .env.example
```

---

## 3. Назначение каталогов

### docs/
Канонические документы проекта.

### apps/
Runnable-приложения (`web`, `api`, `worker`).

### packages/
Переиспользуемые пакеты (`ui`, `config`, `types`).

### tests/
Тесты модулей, интеграций и сценариев.

### scripts/
Скрипты разработки, миграций, обслуживания, проверок.

### deploy/
Документы и артефакты деплоя.

---

## 4. Структура apps/packages

```text
apps/
├─ web/
├─ api/
└─ worker/

packages/
├─ ui/
├─ config/
└─ types/
```

---

## 5. Важное правило

Если файл не укладывается в доменную границу, сначала нужно уточнить архитектуру, а не складывать всё в общий util-модуль без правил.
