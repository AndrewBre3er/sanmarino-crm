# 12. Deployment to VPS

## Статус документа

Этот документ фиксирует только инфраструктурные требования верхнего уровня.
Он не выбирает конкретный runtime или orchestration-стек.

---

## 1. Базовый принцип

Приложение должно быть готово к деплою на VPS, но контекст проекта пока не фиксирует:
- Docker или без Docker
- конкретный reverse proxy
- конкретный application server
- конкретный CI/CD движок

Это должно решаться после выбора прикладного стека.

---

## 2. Что должно быть предусмотрено заранее

Независимо от реализации, деплой должен поддерживать:
- отдельное окружение production
- хранение секретов вне git
- отдельные настройки окружения
- централизованные логи
- резервное копирование БД
- восстановление после сбоя
- health checks
- безопасное обновление без потери данных

---

## 3. Минимальные контуры VPS-архитектуры

Должны быть определены:
- слой приложения
- слой БД
- слой фоновых задач
- слой логов
- слой резервного копирования
- слой reverse proxy / ingress
- слой мониторинга

---

## 4. Обязательные deployment-политики

### 4.1 Secrets
Секреты не хранятся в репозитории.

### 4.2 Backups
Должны быть:
- регулярные бэкапы БД
- проверка восстановления
- политика хранения копий

### 4.3 Migrations
Миграции должны быть контролируемыми и воспроизводимыми.

### 4.4 Rollback
Должен быть описан способ отката.

### 4.5 Observability
Должны быть:
- application logs
- error logs
- базовые метрики доступности
